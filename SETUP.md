## Local Development Setup

### Step 1: Clone Repository and Navigate

```bash
# Navigate to project directory
cd "Human in the loop AI"

# Initialize git (if not already a git repo)
git init
```

### Step 2: Create Python Virtual Environment

```bash
# Create virtual environment
python3 -m venv venv

# Activate virtual environment
# On macOS/Linux:
source venv/bin/activate

# On Windows (PowerShell):
venv\Scripts\Activate.ps1

# On Windows (Command Prompt):
venv\Scripts\activate.bat
```

After activation, your terminal should show `(venv)` prefix.

### Step 3: Install Dependencies

```bash
# Upgrade pip
pip install --upgrade pip

# Install all required packages
pip install -r requirements.txt

# Verify installation
pip list
```

Expected packages:

- fastapi==0.104.1
- uvicorn==0.24.0
- pymongo==4.6.0
- supabase==2.3.5
- pydantic==2.4.2

### Step 4: Create Environment File

```bash
# Copy example env file
cp .env.example .env

# Edit .env with your credentials (see next section)
# macOS/Linux:
nano .env

# Windows PowerShell:
notepad .env

# VS Code:
code .env
```

## Database Configuration

### MongoDB Atlas Setup

1. **Create MongoDB Account**
   - Go to https://www.mongodb.com/cloud/atlas
   - Sign up (free tier available)
   - Create organization and project

2. **Create Free Cluster**
   - Click "Create a Deployment"
   - Choose "Serverless" or "Shared Cluster" (free)
   - Select region (closest to your location)
   - Wait for cluster to be created (5-10 minutes)

3. **Create Database User**
   - Go to "Database Access" in left sidebar
   - Click "Add New Database User"
   - Set username and password (save these!)
   - Select "Autogenerated Secure Password" for strong password
   - Click "Create Database User"

4. **Get Connection String**
   - Go to "Databases" in left sidebar
   - Click "Connect" button on your cluster
   - Choose "Connect your application"
   - Copy the connection string
   - Replace `<username>` and `<password>` with your database user credentials
   - Replace `myFirstDatabase` with `disaster_management`

5. **Add IP Whitelist** (Important!)
   - Go to "Network Access" in left sidebar
   - Click "Add IP Address"
   - Select "Allow Access from Anywhere" (for development)
   - For production, use specific IPs
   - Confirm

6. **Create Database and Collections**
   - In MongoDB Atlas, click "Browse Collections"
   - Create database: `disaster_management`
   - Create collection: `sensor_readings`

7. **Insert Sample Data**

   ```bash
   # Using MongoDB Compass (GUI) or mongosh (CLI)
   # Insert sample sensor readings
   db.sensor_readings.insertMany([
     {
       "barangay_id": 1,
       "water_level": 45.2,
       "location": "Bridge 1",
       "timestamp": new Date("2024-05-03T08:00:00Z")
     },
     {
       "barangay_id": 1,
       "water_level": 48.5,
       "location": "Bridge 1",
       "timestamp": new Date("2024-05-03T08:05:00Z")
     },
     {
       "barangay_id": 1,
       "water_level": 52.1,
       "location": "Bridge 1",
       "timestamp": new Date("2024-05-03T08:10:00Z")
     }
   ])
   ```

8. **Update .env File**
   ```env
   MONGODB_URI=mongodb+srv://username:password@cluster0.xxxxxx.mongodb.net/disaster_management?retryWrites=true
   ```

### Supabase Setup

1. **Create Supabase Account**
   - Go to https://supabase.com
   - Sign up with GitHub or email
   - Create new project

2. **Create Table for Residents**
   - Go to SQL Editor in left sidebar
   - Create new query
   - Run this SQL:

   ```sql
   CREATE TABLE residents (
     id SERIAL PRIMARY KEY,
     barangay_id INTEGER NOT NULL,
     age INTEGER NOT NULL,
     is_pregnant BOOLEAN DEFAULT FALSE,
     is_pwd BOOLEAN DEFAULT FALSE,
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
   );

   -- Create index for faster queries
   CREATE INDEX idx_residents_barangay ON residents(barangay_id);
   ```

3. **Insert Sample Data**

   ```sql
   INSERT INTO residents (barangay_id, age, is_pregnant, is_pwd) VALUES
   (1, 65, false, false),
   (1, 62, false, false),
   (1, 1, false, false),
   (1, 28, true, false),
   (1, 45, false, true),
   (1, 40, false, false),
   (1, 35, false, false);
   ```

4. **Get Connection Credentials**
   - Go to "Settings" → "API" in left sidebar
   - Copy your `Project URL` (SUPABASE_URL)
   - Copy your `anon public` key (SUPABASE_KEY)
   - Never share these keys!

5. **Update .env File**
   ```env
   SUPABASE_URL=https://your-project-id.supabase.co
   SUPABASE_KEY=your-anon-public-key
   ```

## Running Locally

### Start the Development Server

```bash
# Make sure virtual environment is activated
# (venv) should be visible in terminal

# Run the FastAPI server
uvicorn app.main:app --reload --host 0.0.0.0 --port 10000
```

Expected output:

```
INFO:     Uvicorn running on http://0.0.0.0:10000
INFO:     Application startup complete
```

### Access the Application

- **API Base URL**: http://localhost:10000
- **Swagger UI (Interactive Docs)**: http://localhost:10000/docs
- **ReDoc (API Documentation)**: http://localhost:10000/redoc
- **Health Check**: http://localhost:10000/health

### Verify Connection to Databases

The server logs will show:

```
INFO - Successfully connected to MongoDB Atlas
INFO - Successfully connected to Supabase
```

If you see errors, check your `.env` file and database credentials.

## Deployment to Render

### Step 1: Prepare for Deployment

1. **Ensure .gitignore is configured**

   ```bash
   cat > .gitignore << EOF
   venv/
   .env
   __pycache__/
   *.pyc
   .DS_Store
   .vscode/
   .pytest_cache/
   EOF
   ```

2. **Commit your code**

   ```bash
   git add .
   git commit -m "Initial AI Decision Service commit"
   ```

3. **Push to GitHub**
   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/your-repo.git
   git branch -M main
   git push -u origin main
   ```

### Step 2: Create Render Service

1. **Go to Render Dashboard**
   - Visit https://render.com
   - Sign in with GitHub
   - Click on Dashboard

2. **Create New Service**
   - Click "New +" button
   - Select "Web Service"
   - Connect your GitHub repository
   - Search for your repo and select it
   - Click "Connect"

3. **Configure Service**
   - **Name**: `ai-decision-service`
   - **Environment**: `Python 3`
   - **Build Command**:
     ```
     pip install -r requirements.txt
     ```
   - **Start Command**:
     ```
     uvicorn app.main:app --host 0.0.0.0 --port 10000
     ```
   - **Instance Type**: Starter (free)
   - **Region**: Closest to your location

4. **Add Environment Variables**
   - Click "Environment"
   - Add the following variables (get values from your .env):
     - `MONGODB_URI`: Your MongoDB connection string
     - `SUPABASE_URL`: Your Supabase project URL
     - `SUPABASE_KEY`: Your Supabase anon key
     - `ENV`: `production`
     - `PORT`: `10000`

5. **Deploy**
   - Click "Deploy"
   - Wait for deployment to complete (2-5 minutes)
   - Your service will be available at: `https://ai-decision-service.onrender.com`

### Step 3: Verify Deployment

```bash
# Test health check
curl https://ai-decision-service.onrender.com/health

# Test decision endpoint
curl -X POST https://ai-decision-service.onrender.com/api/v1/decision \
  -H "Content-Type: application/json" \
  -d '{"barangay_id": 1}'
```

### Step 4: Monitor Deployment

- **View Logs**: Render Dashboard → Your Service → Logs tab
- **Check Health**: Render Dashboard → Metrics tab
- **Scale if Needed**: Render Dashboard → Settings → Instance Type

## Troubleshooting

### Issue: ModuleNotFoundError

```
ModuleNotFoundError: No module named 'pymongo'
```

**Solution:**

```bash
# Ensure virtual environment is activated
source venv/bin/activate
pip install -r requirements.txt
```

### Issue: MongoDB Connection Refused

```
pymongo.errors.ServerSelectionTimeoutError
```

**Solution:**

1. Check MONGODB_URI in .env file
2. Verify database user credentials
3. Check IP whitelist in MongoDB Atlas (add 0.0.0.0/0)
4. Verify network connectivity

### Issue: Supabase Connection Error

```
ConnectionError: Failed to connect to Supabase
```

**Solution:**

1. Verify SUPABASE_URL and SUPABASE_KEY
2. Check that Supabase project is running
3. Ensure internet connection

### Issue: Port Already in Use

```
OSError: [Errno 48] Address already in use
```

**Solution:**

```bash
# Use a different port
uvicorn app.main:app --port 10001

# Or kill process on port 10000
# macOS/Linux:
lsof -ti:10000 | xargs kill -9

# Windows:
netstat -ano | findstr :10000
taskkill /PID <PID> /F
```

### Issue: Render Deployment Fails

**Check logs:**

1. Go to Render Dashboard
2. Click on your service
3. Go to Logs tab
4. Look for error messages
5. Common issues:
   - Missing environment variables
   - Incorrect Python version
   - Build command failed

**Solution:**

- Fix the issue locally
- Commit and push to GitHub
- Render will automatically redeploy

### Issue: Cold Start on Render

- Free tier services sleep after 15 minutes of inactivity
- First request after sleep may take 10-30 seconds
- Upgrade to paid plan for always-on service
